# 版本发布

## 版本控制说明

由于 Blossom 代码仓库中包含多端代码, 所以并未遵循语义化版本控制，但版本之间仍然具有明确的意义。

- **主版本号**: 如 `v1.x.x`，主版本号的变更代表架构发生可能发生了全新修改、重大功能变更、或者出现了庞大的不兼容修改。

- **次版本号**: 如 `v1.2.x`，次版本号的变更说明发生了服务端代码修改，需要重新部署服务端代码。

- **补丁版本**: 如 `v1.2.1`，补丁版本的变更说明修改发生在客户端或网页端。只需要重新下载客户端或网页端即可。不同补丁版本的客户端使用相同次版本号的服务端。 例如 `v1.2.1` 与 `v1.2.2` 的客户端都使用 `v1.2.0` 版本的服务端。
